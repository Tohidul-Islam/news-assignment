Var is very simple and without restricted. you can declare anywhere and you can get the value from anywhere in the dom.var gets hoisted to the top of the scope.Var have global scope.it can be updated and redeclared.

Const is highly restricted. If you declare a value. you can change it when you will declare with const.Const gets hoisted to the top of the scope but initialize.let have block scope.It can be updated but not redeclared.

if you want to declare a value and somehow you need to change then you should use Let instead of var. Let gets hoisted to the top of the scope but also not initialize. Const have block scope.It can be not updated and not recdeclared.

There are some differences:

a. this value inside a regular function is dynamic and depends on the invocation. But this inside the arrow function is bound lexically and equals to this of the outer function.

b.arguments object inside the regular functions contains the list of arguments. The arrow function, on the opposite, doesn't define arguments (but you can easily access the arrow function arguments using a rest parameter ...args).

c.If the arrow function has one expression, then the expression is returned implicitly, even without using the return keyword.

d.you can define methods using the arrow function syntax inside classes. Fat arrow methods bind this value to the class instance. Anyhow the fat arrow method is invoked, this always equals the class instance, which is useful when the methods are used as callbacks.

Templates Strings are an alternative to string concatenation. They allow variables to be injected into string literals to form fully interpolated strings. They also honor newline characters and other breaks within the string literal.

Template strings are a powerful feature of modern JavaScript released in ES6. It lets us insert/interpolate variables and expressions into strings without needing to concatenate like in older versions of JavaScript. It allows us to create strings that are complex and contain dynamic elements.
